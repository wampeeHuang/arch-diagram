# arch-diagram · 架构图生成器

> 三步流水线：结构定义（节点·边·Phase分组）→ 规则引擎（色板·等宽·间距固化）→ CSS Grid 渲染。

不是"画一个架构图"——是输入数据结构，输出 CSS Grid 渲染的架构图 HTML。颜色、间距、布局全部规则化，不走 AI 自由发挥。

## 设计哲学

### 结构先于视觉
架构图的视觉来自规则引擎，不是AI的审美。先定义节点、边、Phase 分组这些数据，再让规则引擎计算颜色、间距、布局。反过来的流程（先画图再调）永远调不好——因为底层结构没定。

### 规则引擎 = 确定性输出
色板、等宽、间距——全部固化在规则里。同样的数据输入，同样的图输出。确定性是架构图的核心价值——如果两次画出来的不一样，那就不是架构图，是艺术创作。

### CSS Grid 而不是 Canvas
选用 CSS Grid 渲染——文本可选、可搜索、可缩放。Canvas 画出来是图片，Grid 渲染出来是文档。

## 诚实边界

- **适合系统架构/流程架构/部署架构。** 不适合 UI mockup 或信息图
- **节点数 < 100 的场景。** 超大型架构图需要分图
- **输出 HTML，不是 PNG。** 需要图片格式需额外截取
